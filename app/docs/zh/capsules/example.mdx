# 例子：天气 Capsule

这个例子从零搭建一个天气 capsule。最终效果：用户问"北京天气怎么样"，Agent 调用 `RenderCapsule` 工具传入城市名、温度和天气状况，宿主渲染出一张天气卡片。

## 目录结构

一个 capsule 就是一个目录，放在 `~/.jai/capsules/` 下面：

```
~/.jai/capsules/weather/
├── capsule.json    # manifest
├── index.js        # bundle（打包后的产物）
└── src/            # 源码（可选，开发用）
    └── index.tsx
```

## 源码

Capsule 使用 React，通过 `defineCapsule()` 注册。宿主提供 React runtime，capsule 不需要打包 React。

```tsx
import { defineCapsule } from "@jayden/capsule-protocol/runtime";
import { z } from "zod";

export default defineCapsule({
  id: "weather",
  version: "0.1.0",
  components: {
    default: {
      title: "天气",
      description: "展示城市当前天气。当用户询问天气相关问题时使用。",
      dataSchema: z.object({
        city: z.string().describe("城市名，如 Beijing"),
        temp: z.number().optional().describe("摄氏温度"),
        condition: z.string().optional().describe("天气状况短语，如 Sunny、Light rain"),
      }),
      render({ data, theme }) {
        const isDark = theme === "dark";
        const bg = isDark ? "#111418" : "#f8f9fb";
        const fg = isDark ? "#e6e8ec" : "#1d1f23";
        const badgeBg = isDark ? "#2a313a" : "#e7ecf2";

        return (
          <div style={{
            padding: "20px 22px",
            borderRadius: 12,
            display: "flex",
            flexDirection: "column",
            gap: 6,
            background: bg,
            color: fg,
            fontFamily: "-apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif",
          }}>
            <div style={{ display: "flex", alignItems: "center", justifyContent: "space-between" }}>
              <div style={{ fontSize: 13, fontWeight: 500, opacity: 0.7, textTransform: "uppercase" }}>
                {data.city}
              </div>
              <div style={{ fontSize: 11, padding: "2px 8px", borderRadius: 999, background: badgeBg }}>
                {data.condition ?? "—"}
              </div>
            </div>
            <div style={{ display: "flex", alignItems: "baseline", gap: 6 }}>
              <span style={{ fontSize: 48, fontWeight: 700, lineHeight: 1 }}>
                {data.temp ?? "--"}
              </span>
              <span style={{ fontSize: 20, opacity: 0.55, fontWeight: 500 }}>°C</span>
            </div>
          </div>
        );
      },
    },
  },
});
```

几个要点：

- `components` 字段注册组件。这里只有一个组件，key 用 `"default"`
- `dataSchema` 用 zod 定义，`describe()` 的内容会出现在 AI 工具描述里，帮 LLM 理解每个字段的含义
- `render` 接收 `data`（Agent 提供的数据）和 `theme`（宿主当前主题），返回 React 元素
- Capsule 不自己调用 `ReactDOM.render()`——宿主拿到 `render` 函数后负责挂载

## Manifest

`capsule.json` 可以手写，也可以用 `buildCapsuleManifest()` 从 `defineCapsule()` 的返回值自动生成。

手写的话长这样：

```json
{
  "protocol": "capsule/v0",
  "id": "weather",
  "version": "0.1.0",
  "entry": "./index.js",
  "components": {
    "default": {
      "title": "天气",
      "description": "展示城市当前天气。当用户询问天气相关问题时使用。",
      "dataSchema": {
        "type": "object",
        "required": ["city"],
        "properties": {
          "city": {
            "type": "string",
            "description": "城市名，如 Beijing"
          },
          "temp": {
            "type": "number",
            "description": "摄氏温度"
          },
          "condition": {
            "type": "string",
            "description": "天气状况短语，如 Sunny、Light rain"
          }
        }
      }
    }
  }
}
```

几个要点：

- `id` 必须和目录名一致，这里都是 `weather`
- `components` 里每个 key 是一个组件名。单组件 capsule 用 `"default"`
- `dataSchema` 是 JSON Schema 格式。`required` 里只有 `city`，其余可选
- `description` 会被写进 AI 工具的描述里，写清楚什么时候该用这个 capsule

## 打包

Capsule 的 `entry` 指向打包后的 ESM 文件。React 由宿主提供，打包时标记为 external：

```bash
# 用 esbuild 打包
esbuild src/index.tsx --bundle --format=esm --outfile=index.js \
  --external:react --external:react-dom --external:react-dom/client
```

产出的 `index.js` 只包含业务代码，几 KB 大小。

也可以用 `buildCapsuleManifest()` 自动生成 `capsule.json`：

```ts
import { buildCapsuleManifest } from "@jayden/capsule-protocol";
import capsule from "./src/index";

const manifest = buildCapsuleManifest(capsule, { entry: "./index.js" });
// 写入 capsule.json
```

## Agent 看到的

Capsule 安装后，AI 的工具列表里会多一个 `RenderCapsule` 工具。工具描述包含 capsule 信息：

```
Available capsules:

- weather — 天气
  description: 展示城市当前天气。当用户询问天气相关问题时使用。
  data: { city: string (required), temp: number, condition: string }
```

用户问"北京天气怎么样"，Agent：

1. 用其他工具（比如网络搜索）拿到实际天气数据
2. 调用 `RenderCapsule({ capsuleId: "weather", data: { city: "Beijing", temp: 22, condition: "Sunny" } })`
3. 宿主渲染天气卡片，Agent 这轮结束

单组件 capsule 的 `capsuleId` 就是 capsule 的 `id`，不需要指定 component。

## 多组件的例子

一个 capsule 可以注册多个组件。比如一个 finance capsule：

```tsx
export default defineCapsule({
  id: "finance",
  version: "0.1.0",
  components: {
    "stock-chart": {
      title: "K线图",
      description: "展示股票价格走势。当用户询问某只股票行情时使用。",
      dataSchema: z.object({
        symbol: z.string().describe("股票代码"),
        prices: z.array(z.object({
          date: z.string(),
          close: z.number(),
        })).describe("历史收盘价序列"),
      }),
      render({ data, theme }) {
        return <StockChart symbol={data.symbol} prices={data.prices} theme={theme} />;
      },
    },
    "portfolio": {
      title: "持仓概览",
      description: "展示当前持仓分布。当用户询问持仓情况时使用。",
      dataSchema: z.object({
        holdings: z.array(z.object({
          name: z.string(),
          value: z.number(),
          percent: z.number(),
        })),
      }),
      render({ data, theme }) {
        return <PortfolioView holdings={data.holdings} theme={theme} />;
      },
    },
  },
});
```

Agent 看到的工具描述：

```
- finance/stock-chart — K线图
  description: 展示股票价格走势。当用户询问某只股票行情时使用。
  data: { symbol: string (required), prices: Array<{ date: string, close: number }> (required) }

- finance/portfolio — 持仓概览
  description: 展示当前持仓分布。当用户询问持仓情况时使用。
  data: { holdings: Array<{ name: string, value: number, percent: number }> (required) }
```

调用时 `capsuleId` 用 `"finance/stock-chart"` 格式，斜杠前是 capsule id，斜杠后是 component key。

## 主题适配

`render` 的 `theme` 参数告诉 capsule 当前是亮色还是暗色。上面的例子根据 `isDark` 切换了背景色和文字颜色。

不是必须的，但做了主题适配看起来会更协调。
