# 协议规范

Capsule Protocol 定义四件事：capsule 如何描述自己（manifest）、代码如何打包（bundle）、如何注册组件（defineCapsule）、运行时如何通信（wire messages）。

协议不绑定特定宿主。iframe、WebView、独立进程都可以。

## Manifest

每个 capsule 有一个 `capsule.json`：

```ts
interface CapsuleManifest {
  protocol: "capsule/v0";
  id: string;
  version: string;
  entry: string;
  components: Record<string, {
    title?: string;
    description?: string;
    dataSchema: JSONSchema;
  }>;
  _meta?: Record<string, unknown>;
}
```

| 字段 | 必填 | 说明 |
|------|------|------|
| `protocol` | 是 | 固定值 `"capsule/v0"` |
| `id` | 是 | 唯一标识符，非空字符串 |
| `version` | 是 | 建议 SemVer |
| `entry` | 是 | ESM bundle 路径，相对于 manifest 所在目录 |
| `components` | 是 | 组件注册表，key 是组件名 |
| `_meta` | 否 | 扩展字段，协议不解析 |

### components

每个 component 有独立的 `dataSchema` 和可选的 `title`、`description`。

单组件 capsule 用 `"default"` 作为 key：

```json
{
  "components": {
    "default": {
      "title": "天气",
      "description": "展示城市当前天气",
      "dataSchema": { "type": "object", "properties": { "city": { "type": "string" } } }
    }
  }
}
```

多组件 capsule 用有意义的 key：

```json
{
  "components": {
    "stock-chart": {
      "title": "K线图",
      "dataSchema": { ... }
    },
    "portfolio": {
      "title": "持仓概览",
      "dataSchema": { ... }
    }
  }
}
```

`description` 会写进 AI 工具描述。写清楚什么时候该用这个组件，LLM 才能准确判断。

### 前向兼容

校验器遇到 manifest 中的未知字段时应该忽略而不是拒绝。`_meta` 的内容视为不透明数据。

## Bundle

`entry` 指向一个自包含的 ESM 文件。

要求：

- 所有业务依赖打包在内——沙箱没有模块加载器
- **React 不打包**。宿主提供 React runtime，打包时将 `react`、`react-dom`、`react-dom/client` 标记为 external
- 必须有 default export，值是 `defineCapsule()` 的返回值
- 可通过 `import()` 动态加载

```bash
esbuild src/index.tsx --bundle --format=esm --outfile=index.js \
  --external:react --external:react-dom --external:react-dom/client
```

## 组件注册

Capsule 通过 `defineCapsule()` 注册组件。这个函数不执行渲染，只返回一个注册表对象。

```ts
import { defineCapsule } from "@jayden/capsule-protocol/runtime";
import { z } from "zod";

export default defineCapsule({
  id: "weather",
  version: "0.1.0",
  components: {
    default: {
      dataSchema: z.object({ city: z.string() }),
      render({ data, theme }) {
        return <div>{data.city}</div>;
      },
    },
  },
});
```

### defineCapsule 签名

```ts
function defineCapsule(config: {
  id: string;
  version: string;
  components: Record<string, {
    title?: string;
    description?: string;
    dataSchema: z.ZodType;
    render: (props: CapsuleProps) => ReactElement | null;
  }>;
}): CapsuleRegistry;
```

### CapsuleProps

```ts
interface CapsuleProps<D = unknown> {
  data: D;
  theme: "light" | "dark" | null;
  instanceId: string;
}
```

- `data` — Agent 提供的数据，类型由 `dataSchema` 推断
- `theme` — 宿主当前主题，`null` 表示宿主没有提供
- `instanceId` — 实例标识符

### render 函数

- 返回 React 元素
- 纯函数，不应有副作用
- 不需要调用 `ReactDOM.render()`——宿主负责挂载

### 宿主如何使用注册表

宿主加载 bundle 后的流程：

1. 通过 dynamic import 拿到 default export（`CapsuleRegistry`）
2. 在 `registry.components` 中按 key 找到目标 component
3. 调用 `component.render({ data, theme, instanceId })` 拿到 React 元素
4. 用 `createRoot(element).render(...)` 挂载

Capsule 代码不需要感知宿主的加载机制。

## Wire messages

运行时通信只有三种消息，全部从沙箱发往宿主。传输方式不做规定（postMessage、IPC 都行），协议只定义消息结构。

### capsule/ready

```ts
{ type: "capsule/ready", instanceId: string }
```

Bundle 加载完毕，首次渲染完成。可选——宿主应该在没收到这条消息时也能正常工作。

### capsule/error

```ts
{ type: "capsule/error", instanceId: string, message: string, stack?: string }
```

未捕获的异常或未处理的 Promise rejection。

### capsule/resize

```ts
{ type: "capsule/resize", instanceId: string, width?: number, height?: number }
```

根元素尺寸变化。宿主可以据此调整容器大小。可选。

### 通用规则

- 每条消息都带 `instanceId`，必须和宿主分配的值一致
- 宿主收到未知 `instanceId` 的消息应丢弃
- 宿主收到未知 `type` 的消息应忽略（前向兼容）
- 不存在宿主→沙箱方向的消息

## 安全指导（非规范性）

协议建议宿主：

- 在 null-origin 沙箱中加载 capsule（如 `<iframe sandbox="allow-scripts">`）
- 对沙箱施加严格的 Content Security Policy
- 在发送数据给 capsule 之前用 `dataSchema` 校验
- 隔离 capsule 实例——不共享全局状态

Capsule 不应假设自己能访问网络或存储。外部数据应由 Agent 事先获取，通过 `data` 传入。

## 渲染模型

Capsule 是一次性渲染。

数据在启动时注入一次，capsule 渲染一次，结束。没有数据更新机制，没有双向通信，没有 dispose 生命周期。

要展示新数据，Agent 重新调用工具，宿主创建新的 capsule 实例。

## 版本

当前协议标识符：`capsule/v0`。不稳定，可能有破坏性变更。

`capsule/v1`（未来）会承诺大版本内向后兼容：

- 新增字段不算破坏
- 移除或改变字段语义是破坏——升大版本
- 宿主必须忽略不认识的字段和消息类型
