# Example: weather capsule

This example builds a weather capsule from scratch. End result: when a user asks "what's the weather in Beijing," the agent calls `RenderCapsule` with city name, temperature, and conditions, and the host renders a weather card.

## Directory structure

A capsule is a directory under `~/.jai/capsules/`:

```
~/.jai/capsules/weather/
├── capsule.json    # manifest
├── index.js        # bundle (build output)
└── src/            # source (optional, for development)
    └── index.tsx
```

## Source code

Capsules use React, registered via `defineCapsule()`. The host provides the React runtime — capsules don't bundle React.

```tsx
import { defineCapsule } from "@jayden/capsule-protocol/runtime";
import { z } from "zod";

export default defineCapsule({
  id: "weather",
  version: "0.1.0",
  components: {
    default: {
      title: "Weather",
      description: "Show current weather for a city. Use when the user asks about weather.",
      dataSchema: z.object({
        city: z.string().describe("City name, e.g. Beijing"),
        temp: z.number().optional().describe("Temperature in Celsius"),
        condition: z.string().optional().describe("Short weather phrase, e.g. Sunny, Light rain"),
      }),
      render({ data, theme }) {
        const isDark = theme === "dark";
        const bg = isDark ? "#111418" : "#f8f9fb";
        const fg = isDark ? "#e6e8ec" : "#1d1f23";
        const badgeBg = isDark ? "#2a313a" : "#e7ecf2";

        return (
          <div style={{
            padding: "20px 22px",
            borderRadius: 12,
            display: "flex",
            flexDirection: "column",
            gap: 6,
            background: bg,
            color: fg,
            fontFamily: "-apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif",
          }}>
            <div style={{ display: "flex", alignItems: "center", justifyContent: "space-between" }}>
              <div style={{ fontSize: 13, fontWeight: 500, opacity: 0.7, textTransform: "uppercase" }}>
                {data.city}
              </div>
              <div style={{ fontSize: 11, padding: "2px 8px", borderRadius: 999, background: badgeBg }}>
                {data.condition ?? "—"}
              </div>
            </div>
            <div style={{ display: "flex", alignItems: "baseline", gap: 6 }}>
              <span style={{ fontSize: 48, fontWeight: 700, lineHeight: 1 }}>
                {data.temp ?? "--"}
              </span>
              <span style={{ fontSize: 20, opacity: 0.55, fontWeight: 500 }}>°C</span>
            </div>
          </div>
        );
      },
    },
  },
});
```

Key points:

- `components` registers components. This capsule has one component with key `"default"`
- `dataSchema` is defined with zod. The `describe()` text appears in the AI tool description, helping the LLM understand each field
- `render` receives `data` (from the agent) and `theme` (host's current theme), returns a React element
- The capsule doesn't call `ReactDOM.render()` itself — the host takes the `render` function and handles mounting

## Manifest

`capsule.json` can be hand-written or auto-generated from `defineCapsule()`'s return value using `buildCapsuleManifest()`.

Hand-written:

```json
{
  "protocol": "capsule/v0",
  "id": "weather",
  "version": "0.1.0",
  "entry": "./index.js",
  "components": {
    "default": {
      "title": "Weather",
      "description": "Show current weather for a city. Use when the user asks about weather.",
      "dataSchema": {
        "type": "object",
        "required": ["city"],
        "properties": {
          "city": {
            "type": "string",
            "description": "City name, e.g. Beijing"
          },
          "temp": {
            "type": "number",
            "description": "Temperature in Celsius"
          },
          "condition": {
            "type": "string",
            "description": "Short weather phrase, e.g. Sunny, Light rain"
          }
        }
      }
    }
  }
}
```

Key points:

- `id` must match the directory name — both are `weather` here
- `components` maps component keys to definitions. Single-component capsules use `"default"`
- `dataSchema` is JSON Schema format. Only `city` is required, the rest are optional
- `description` gets written into the AI tool description — be specific about when to use this capsule

## Building

The capsule's `entry` points to the bundled ESM file. React is provided by the host, so mark it as external when bundling:

```bash
# Bundle with esbuild
esbuild src/index.tsx --bundle --format=esm --outfile=index.js \
  --external:react --external:react-dom --external:react-dom/client
```

The output `index.js` contains only business code — typically a few KB.

You can also use `buildCapsuleManifest()` to auto-generate `capsule.json`:

```ts
import { buildCapsuleManifest } from "@jayden/capsule-protocol";
import capsule from "./src/index";

const manifest = buildCapsuleManifest(capsule, { entry: "./index.js" });
// write to capsule.json
```

## What the agent sees

After the capsule is installed, the AI's tool list includes a `RenderCapsule` tool. The tool description includes capsule info:

```
Available capsules:

- weather — Weather
  description: Show current weather for a city. Use when the user asks about weather.
  data: { city: string (required), temp: number, condition: string }
```

When a user asks "what's the weather in Beijing," the agent:

1. Uses other tools (e.g. web search) to get actual weather data
2. Calls `RenderCapsule({ capsuleId: "weather", data: { city: "Beijing", temp: 22, condition: "Sunny" } })`
3. The host renders the weather card, agent's turn ends

For single-component capsules, `capsuleId` is just the capsule's `id` — no need to specify a component.

## Multi-component example

A capsule can register multiple components. For example, a finance capsule:

```tsx
export default defineCapsule({
  id: "finance",
  version: "0.1.0",
  components: {
    "stock-chart": {
      title: "Stock Chart",
      description: "Show stock price trends. Use when the user asks about a stock.",
      dataSchema: z.object({
        symbol: z.string().describe("Stock ticker symbol"),
        prices: z.array(z.object({
          date: z.string(),
          close: z.number(),
        })).describe("Historical closing price series"),
      }),
      render({ data, theme }) {
        return <StockChart symbol={data.symbol} prices={data.prices} theme={theme} />;
      },
    },
    "portfolio": {
      title: "Portfolio",
      description: "Show current portfolio allocation. Use when the user asks about holdings.",
      dataSchema: z.object({
        holdings: z.array(z.object({
          name: z.string(),
          value: z.number(),
          percent: z.number(),
        })),
      }),
      render({ data, theme }) {
        return <PortfolioView holdings={data.holdings} theme={theme} />;
      },
    },
  },
});
```

The agent sees:

```
- finance/stock-chart — Stock Chart
  description: Show stock price trends. Use when the user asks about a stock.
  data: { symbol: string (required), prices: Array<{ date: string, close: number }> (required) }

- finance/portfolio — Portfolio
  description: Show current portfolio allocation. Use when the user asks about holdings.
  data: { holdings: Array<{ name: string, value: number, percent: number }> (required) }
```

The `capsuleId` uses `"finance/stock-chart"` format — everything before the slash is the capsule id, after is the component key.

## Theme support

The `render` function's `theme` parameter indicates whether the host is in light or dark mode. The example above switches background and text colors based on `isDark`.

Not required, but capsules with theme support look better integrated.
