# Impeccable — Design Context for OpenPanda

> This file is the single source of truth for OpenPanda's design direction. All `/impeccable`, `/craft`, `/layout`, `/typeset`, `/colorize`, etc. skills MUST read this file before producing any visual work. Update via `/impeccable teach`.

---

## Product

**OpenPanda** — an open-source desktop AI coding companion. Positioned as "Claude Code, upgraded" but deliberately **NOT** a code editor. The desktop face of the [OpenClaw](https://docs.openclaw.ai/) ecosystem: a 24/7-available local + cloud agent you talk to, not an IDE you configure.

- **Repo**: open source (visual identity must survive being screenshot on a GitHub README)
- **Stack**: Electron + React 19 + Tailwind 4 + shadcn/ui (`radix-nova`, neutral base) + Base UI + Radix + `motion` + `streamdown` + next-themes
- **Related projects**: OpenClaw (CLI/gateway, self-hosted agent across messaging channels). OpenPanda is the desktop client form of this same family.

## Design Context

### Users

- **Primary**: developers and technical power users who want a personal AI assistant always-on at their desk, not only inside an editor. They already use Claude Code / Cursor / Claude Desktop and find them either too "IDE-shaped" or too "chatbot-shaped."
- **Context of use**: 24-hour companion. Ambient. Kept open on a second monitor, on the dock, or summoned quickly. Used across the full day — morning planning, coding sessions, after-hours "just thinking out loud" with the model.
- **Job-to-be-done**: have a trusted, always-there collaborator for thinking, coding, and getting things done. Not "execute a command"; more like "talk to the thoughtful colleague across the desk."

### Brand Personality

Three words (provisional — refine during `/impeccable teach` re-runs):

1. **Gentle (温柔)** — never shouts. No aggressive CTAs, no neon, no urgency theater.
2. **Crafted** — every detail looks hand-tuned, not generated. Like letterpress, not laser-cut. Closer to a bookshop than a SaaS dashboard.
3. **Alive** — has subtle personality via the panda mascot, the breathing cursor, and micro-motion. Not cold-professional.

Emotional goal: users should feel **calm, capable, and not alone** when OpenPanda is open. The product should feel like a warm lamp on a desk at night, not a control panel.

### Aesthetic Direction

**Aesthetic target** — warm off-white surfaces (never pure white), restrained serif accents at identity moments, humanist sans for everything else, generous breathing room, whispered shadows instead of hard drop-shadows, and a single warm accent used sparingly. The product should feel like a well-designed notebook someone chose to open — not a panel that demanded attention.

Think physical objects, not design systems: a hardcover notebook on a wooden desk, a mid-century lamp, a paperback novel, the interior of a small specialty bookshop, the menu at a quiet neighborhood café. Reject anything that would look at home in a startup pitch deck.

**Visual DNA (already established in code — keep and strengthen):**

| Token | Value | Notes |
|-------|-------|-------|
| Brand accent | `#546448` (light) / `#9aaa82` (dark) — `--primary-2` | Moss green. Used for links, mascot context, chat cursor glow. Do NOT replace. |
| Primary CTA | near-black / near-white | Hierarchy: black button = commit action; moss underline = editorial link. |
| Neutral palette | Olive-tinted OKLCH, hue ~106–115, low chroma (0.001–0.039) | Never pure gray. Carries moss into the surfaces themselves. |
| Radius | `--radius: 1rem` with scaled sm/md/lg/xl/2xl | Soft but not pillowy. Stays. |
| Mascot | 🐼 panda SVG (`panda-1.svg`, `panda-2.svg`) | Core asset. Gets more life, not less — micro-animations, expression variants, ambient idle states are welcome. |
| Motion | spring, stiffness 260–300, damping 20–24 | Current feel is correct. Extend this language; don't swap it for ease-out-cubic. |
| Custom chat cursor | moss-colored breathing glow | Keep. Exemplar of "alive" personality. |

**Typography (committed):**

- **UI sans / body**: **Hanken Grotesk Variable** (weight 100–900). Humanist sans with warm, underused personality. Covers all UI chrome, body text, and `font-heading` (which aliases to sans — hierarchy is built via weight/size, not family).
- **Display serif / brand**: **Literata Variable** (weight 200–900 + optical size axis). Designed for long-form reading; tightens at body size, gains sculpted weight at display size via `font-optical-sizing: auto`. Used exclusively for identity moments.
- **Mono / code**: system stack (`ui-monospace, SFMono-Regular, Menlo, ...`). Mono is never brand identity in OpenPanda — per Design Principle 5 (Editorial > dashboard).
- **CJK (中文)**: system fallback — macOS `PingFang SC` (sans) / `Songti SC` (serif); Windows `Microsoft YaHei` / `Noto Serif CJK SC`. Do not self-host CJK fonts — OS-provided families are excellent and shipping multi-MB CJK variable fonts is not justified.

**Usage rules:**

- `font-sans` / `font-heading` → Hanken Grotesk. Default 400 for body, 500 for headings/labels, 600 for emphasis.
- `font-serif` → Literata. Used in **4 identity locations only** at initial rollout: `EmptyState` logotype (`OpenPanda` hero), `AppToolbar` title, `ChatHeader` title (edit + display), and `CompactionDivider` labels (as `italic` for book-chapter-break feel). Every new `font-serif` usage must be justified as an identity or editorial moment — never decoration.
- `font-mono` → system stack only. Used for: code blocks, model IDs, API key inputs. Not UI chrome.
- `font-synthesis: none` is set globally — never rely on synthetic bold or italic. Use real variable weight axes.

**Rejected (off the table):** Inter, DM Sans, Space Grotesk, Plus Jakarta Sans, IBM Plex (all variants), Fraunces, Newsreader, Lora, Crimson, Playfair Display, Cormorant, Syne, Outfit, Instrument Sans, Instrument Serif, Geist (overused-adjacent). See `impeccable/SKILL.md` `reflex_fonts_to_reject` for the canonical list. Any future reconsideration goes through `/typeset` again, not ad-hoc swaps.

**Color / Theme:**

- Support both light and dark **with equal care**. No "dark is the cool default, light is an afterthought" and no "light is safe."
- Light mode is the **narrative default** — matches the "desk lamp at night" and "bookshop" feeling better than black-on-black.
- Dark mode must preserve warmth — never fall into `oklch(0.2 0 0)` pure-gray IDE darkness. Keep the olive hue tint all the way into shadows.
- Moss green is the ONLY chromatic accent in the system. Charts may use the moss-based chart-1 through chart-5 ramp already defined. Never introduce a second accent hue without re-doing teach.

### Anti-References (what OpenPanda must NEVER look like)

Explicit rejects — if output starts drifting toward these, reset:

- **Cursor**: dense IDE chrome, purple gradient mark, "tool for pros" coldness. OpenPanda is not that.
- **VS Code / Zed / any editor**: panel-and-sidebar-as-identity, monospace in UI chrome, blue/purple activity bars. OpenPanda is **not a code editor**.
- **Claude Desktop / ChatGPT Desktop**: orange or green brand accent, Söhne-as-default, modal-centric flows, "enterprise chatbot" air.
- **Warp / Raycast**: command-palette-as-hero, mono everywhere, "power user" signaling via glyph density.
- **Linear / Vercel / Notion 2024**: sharp geometric sans + neutral gray + thin borders. The generic "modern SaaS" monoculture.
- **Any AI product with**: cyan-on-dark gradients, purple→blue brand mark, glassmorphism, "made with GPT" visual clichés.

### Design Principles

These five principles override personal taste during any design decision:

1. **Calm over clever.** If a flourish earns a second look but doesn't earn a third, cut it. OpenPanda is used all day; overdesign fatigues.
2. **Mascot as companion, not logo.** The panda appears with breathing, idle blinks, states, and moods — not as a static brand mark in a corner. Every mascot appearance is a small animation opportunity. But: never cute-for-cute's-sake; the panda is calm, competent, observing.
3. **Chat is the primary surface.** Not panels, not sidebars, not command palettes. Whatever we add, the chat column remains the room; everything else is furniture.
4. **Light is home.** Light mode is the canonical visual. Dark mode is a respected alternate. Dark-first features are the exception, not the default.
5. **Editorial > dashboard.** Typography hierarchy, reading comfort, generous leading, restrained color. Closer to a good reading app than a good admin UI. No stat cards, no metric tiles, no sparklines-as-decoration.

### Language / Localization

- UI is currently **bilingual (中文 + English)** and this is intentional — do not homogenize to one language without asking. Chinese is used for affective/user-facing copy (`对话记录`, `暂无对话记录`, `重命名`, `删除`), English for product nouns and developer-facing words (`New Chat`, `OpenPanda`, `Settings`).
- Heading types should render both Chinese and Latin script well. This further pushes away from geometric display serifs that only look good in Latin.

### Accessibility (provisional — confirm in future teach)

- Target WCAG AA minimum. Moss green on moss-tinted cream must be validated per component.
- Respect `prefers-reduced-motion`. The mascot idle animation and cursor breathing should downgrade to static when reduced motion is on — keep the moss color, drop the movement.
- No color-only state signaling. Pair with weight, position, or icon.

## Open Questions (resolve before major work)

These were deferred during the first teach pass and should be answered when they become blocking:

- **Mascot expression system.** Does the panda have 2 poses, or a small state machine (idle / thinking / error / celebrate)? Decide before the next empty-state / loading-state redesign.
- **Settings / dense-form surfaces.** Settings today sits uncomfortably between "editorial" and "form UI." Decide whether settings is an exception to the editorial rule, or whether it needs a domain-specific subsystem.
- **Marketing site vs. product app.** The README and any future docs.openpanda.ai must share this DNA. A second teach pass may be needed when the marketing surface appears.
